    R에서 RImpala 패키지를 이용해 Impala 커넥션 맺기 (with Docker)

    21 Mar 2017 » R


    Impala 라는걸 사용해 봐야 할 일이 생겼다.
    필자는 Hadoop, SQL 과 같은 기술이 상당히 낯설고 두려움이 있었는데 Impala 를 통해 조금씩 알게 되는 계기가 되었으면 좋겠다.

    Impala 는 아파치 프로젝트에 incubating 단계로 관리되고 있는 프로젝트로써
    HDFS(Hadoop Distributed File System) 안에 있는 데이터를 SQL 을 통해 실시간으로(대화형으로) 분석할 수 있게 도와주는 시스템이라고 한다.
    HDFS 를 이용하기 위해선 MapReduce 프레임워크를 알고 이용했어야 한다는 불편한 제약을 Impala 를 통해 벗어날 수 있다 이해할 수 있겠다.

    Impala 소개는 이쯤 하고 (읔.. 알고있는게 이정도 뿐이다 😅) 여하튼 지금은 Docker 을 통해 Impala 를 간단히 구동시켜보고,
    RImpala package 를 이용해 R 상에서 Impala 와 커넥션을 맺는데 성공했던 과정을 짤막하게 정리하고 적어보고자 한다.



    Ready impala

    Impala 이용 전 .jar 리소스들을 준비해야 한다고 한다.
    아래 R 코드를 통해 다운로드 및 워킹디렉토리에 압축해제가 가능하다.

    download.file("https://github.com/Mu-Sigma/RImpala/blob/master/impala-jdbc-cdh5.zip?raw=true", 
              "jdbc.zip", method = "curl", extra = "-L")
unzip("jdbc.zip")



    Docker images ready

    그다음 터미널을 들어가 Docker 를 구동할 준비를 한다.
    여기서 Docker 를 이용하려는 이유를 언급해야 할 것 같은데
    Impala 를 이용하기 위해선 Hadoop 시스템이 사전에 준비되어야 하고, JDBC 를 위한 Java 설정등 다양한 준비(Requirements)가 매우 부담되기 때문이다.1
    이러한 이유로 나는 Docker 를 이용해 누군가 개발해 놓은, 사전에 잘 구축한 환경을 사용하고자 하기 때문에
    Docker 구동을 위해 boot2docker 를 시작했다.

    $ boot2docker init
$ boot2docker up
$ boot2docker ssh

    boot2docker ssh 를 통해 docker 가상머신에 접속 후 "Impala" 를 검색어로 적당한 이미지를 찾아본다.

    여러 가지가 이미지가 준비되어 있는데 그중 개인적으론 "cloudera/impala-dev" 보단 "codingtony/impala" 가 바로 사용하기 좋았다.
    "codingtony/impala" 이미지를 준비 내려받았다.

    docker@boot2docker:~$ docker pull codingtony/impala

    1.4GB 정도가 다운로드 되는데 시간이 조금 소요된다.
    다운로드 이후 image 가 잘 준비되어 있는지 docker images 를 통해 확인할 수 있다.

    docker@boot2docker:~$ docker images
    REPOSITORY          TAG                 IMAGE ID            CREATED             VIRTUAL SIZE
codingtony/impala   latest              24f0d50712cf        2 years ago         1.452 GB

    다음을 통해 준비한 Impala 이미지를 컨테이너로 실행시킨다.2

    docker@boot2docker:~$ docker run -d --name "impala" -p 9000:9000 -p 50010:50010 -p 50020:50020 -p 50070:50070 -p 50075:50075 -p 21000:21000 -p 21050:21050 -p 25000:25000 -p 25010:25010 -p 25020:25020 codingtony/impala

    참고로 docker run 의 다양한 옵션들 중 -d, --name 은 각각

    • -d : background 프로세스로 실행해 놓겠다는 의미
    • --name "impala" : image 이름의 alias 를 부여, 예를 들어 본 컨테이너에 접근하기 위해선 "codingtony/impala" 의 이름을 다 입력해야 하는데 앞으론 "impala" 라는 이름으로 접근 가능

    의 의미를 가지고 있다.
    참고로 Impala 컨테이너가 백그라운드로 잘 실행되는 상태인지 확인을 위해선 docker ps 명령어로 프로세스 확인할 수 있다.
    프로세스가 잘 돌고 있는 것을 확인했다면

    docker@boot2docker:~$ docker exec -it impala /bin/bash

    를 통해 Impala 컨테이너의 bash 셸을 열어 입력/출력이 가능하도록 사용할 수 있고
    impala-shell 를 실행해 대화형 쿼리를 주고받을 수 있다.

    RImpala package 이용한 로컬에서의 impala 제어

    Docker VM IP 확인

    로컬 상에서 RImpala package 이용한 로컬 상의 제어를 위해 docker ip 를 확인할 필요가 있다.
    이는 boot2docker ip 를 통해 쉽게 확인 가능하다.

    $ boot2docker ip

    예를 들어 "192.168.59.103" 임을 위의 커멘드를 통해 확인했다고 가정하겠다.

    R상에 RImpala 패키지를 이용한 커넥션

    R로 되돌아오자.
    먼저 .jar 들을 다운받은 디렉토리 path를 rimpala.init() 함수에 입력한 코드를 먼저 실행하고,
    docker 가상환경의 ip, 그리고 Impala JDBC/ODBC 기본값 포트번호 (21050) 를 rimpala.connect() 에 입력한 코드를 그다음으로 실행하면 TRUE 가 반기면 커넥션이 성공한 것이다.

    library(RImpala)

rimpala.init(libs = "impala-jdbc-cdh5")
    ## [1] "Classpath added successfully"
    rimpala.connect(IP = "192.168.59.103", port = "21050")
    ## [1] TRUE

    Use databases

    자 이제 즐기면(?) 되겠다. database 가 무엇이 있는지 확인부터 해보자.
    show databases; 쿼리를 날려보는 건 rimpala.showdatabases() 을 실행하는 것과 같다.

    rimpala.showdatabases()
    ##               name
## 1 _impala_builtins
## 2          default

    쿼리문 자체를 이용하고자 한다면 rimpala.query() 함수를 이용하면 된다.
    단 쿼리문 마지막에 세미콜론(;)은 넣지 않아야 에러가 나지 않았다.

    rimpala.query("show databases")
    ##               name
## 1 _impala_builtins
## 2          default

    사용할 데이터베이스 선택을 해보자.
    "default" database 를 사용하려한다면

    rimpala.usedatabase("default")
    ## [1] TRUE

    를 하면된다. use default; 쿼리와 같다.

    Send query test

    tryCatch(rimpala.query("drop table sample"), error = function(e){})
    ## NULL
    tryCatch(rimpala.query("create table sample(no integer)"), error = function(e){})
    ## NULL
    tryCatch(rimpala.query("insert into sample (no) values (1)"), error = function(e){})
    ## NULL
    data <- rimpala.query("select * from sample") # output value is data.frame class
data
    ##   no
## 1  1

    rimpala.query() 함수로 쿼리를 보낼 때 에러를 많이 반환했다.
    하지만 이상하게도 R상에선 에러를 뿜었으나 쿼리가 Impala 에 잘 먹혔다.
    이 때문에 임시방편으로 tryCatch() 를 통해 예외처리를 했다.
    다만 데이터를 끌어오는데는 쿼리는 문제없이 잘 실행됬다.

    close connect impala

    커넥션은 rimpala.close() 로 끊을 수 있다.

    rimpala.close()
    ## [1] TRUE
