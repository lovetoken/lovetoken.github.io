    R에서 `try()` 함수 소개

    11 Feb 2016 » R

    이전에 나는 특정 코드들이 있을 때, 이 코드를 수행하면 에러가 나는지, 나지 않는지 여부를 boolean 값으로 반환하는 그러한 함수를 찾고 있었다. (그보단 이러한 함수를 만들 수 있도록 리소스를 찾아보고 있었다.)
    그러던 중 우연히 알게 된 try() 함수에 대해 알아보고자 한다.

    아래 코드는 bsid 라는 environment 객체가 없기 때문에 아래의 코드는 에러를 내어 본 파일의 rmarkdown 랜더링을 막게 될 것이다.
    우선 랜더링(컴파일링)마저 막는 상황을 피하기 위해 코드청크를 eval=F 로 설정했다.

    ls(envir=bsdi)

    실제로 bsid 라는 environment 객체가 없는 상황에 위 코드를 실행시키면

    > ls(envir=bsdi)
Error in ls(envir = bsdi) : object 'bsdi' not found

    의 에러메세지가 출력된다.

    try() function

    try(expr={
  ls(envir=bsdi)
})

    try() 함수 안에는 표현식이 들어가는데 이전 예제 코드의 ls(envir=bsdi) 를 입력하였다.
    ls(envir=bsdi) 코드는 역시 에러가 반환된다.
    하지만 차이점은 에러를 반환하고도 이후 코드를 계속적으로 수행한다는 차이점을 가지고 있다.
    무슨소리인지 잘 모르겠다면 아래코드를 보면 된다.

    try(expr={
  ls(envir=bsdi)
})

print("Hello R")

    ## [1] "Hello R"

    try() 함수를 쓴 코드 이후의 코드인 print("Hello R") 은 정상적으로 실행되 수행된 것을 볼 수 있다.
    유의할 것은 실제로 위 코드청크에 eval=F 설정을 주지 않고도 렌더링이 막히지 않았는다는 것이다.
    따라서 에러를 뿜는 코드이더라 하더라도, 그것이 try() 함수로 감싸지면 계속적으로 수행된다는 것이다.

    silent 인자

    try() 함수의 silent 인자로 command 에 에러가 반환되는 메세지 조차 침묵시킬 수 있다.
    이 인자의 값을 TRUE 로 설정하면 에러는 silent 시킨다.

    try(expr={
  ls(envir=bsdi)
}, silent=T)

    차이점을 체감하기 위해선 직접 R command 를 수행해볼 필요가 있다.

    두 줄 의 코드를 try 하되 한줄은 정상, 한줄은 에러를 내는 코드로 수정하면?

    try(expr={
  print("Hello R")   # normal code
  ls(envir=bsdi)     # error code
}, silent=T)

    ## [1] "Hello R"

    첫번째 코드는 정상적으로 문자열 “Hello R” 을 출력시킨다.
    그리고 에러반환 코드 2번째 코드는 반응이 없다.

    순서를 바꾸어 보면?

    try(expr={
  ls(envir=bsdi)     # error code
  print("Hello R")   # normal code
}, silent=T)

    헛.. 첫번째 코드가 에러를 뿜기 때문에 그 이후 코드인 print("Hello R")는 수행되지 않는것을 볼 수 있다.
