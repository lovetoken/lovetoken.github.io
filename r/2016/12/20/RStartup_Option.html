    R에서 Startup option 설정하기 (on Mac OSX)

    20 Dec 2016 » R

    만약 R에서 꾸준히 + 자주자주 + 빈번하게 활용하는 패키지가 있다면 매번 library() 로 호출하는 것이 귀찮을 수 있다.
    나의 경우 dplyr package 를 항시 사용하는 편이어서 R이 켜질 때 자동으로 library(dplyr) 이 실행됐으면 했는데 찾아보니 R은 이런 경우를 생각해 Startup option 들을 제공하고 있다는 것을 알았다.

    여러 방법이 있지만 내가 볼 때 가장 간편한 방법은
    자신의 홈 디렉토리에 ".Rprofile" 이란 숨김파일을 만든 후
    이안에 자주 활용하는 패키지를 호출하는 명령어를 미리 적어 놓는 것이다.

    자세히 정리하면 아래와 같다.

    A) 홈 디렉토리로 들어가 ".Rprofile" 숨김파일 만들기

    쉘에서 틸드(~)문자는 사용자의 홈 디렉토리를 의미하는 특별한 기호이다.
    터미널을 실행한 후 틸드문자를 이용해 사용자 홈디렉토리로 이동한다.

    cd ~

    아마도 ".Rprofile" 파일이 존재할 수도 있다. 물론 없을 수도 있고 상황에 따라 다르다.
    상관없이 텍스트에디터를 이용해 ".Rprofile" 를 수정하거나 새로 만들면 되겠다.
    간편히 nano 를 이용한다고 치면1

    nano .Rprofile

    를 통해 ".Rprofile" 를 열거나 새로 만들게 된다.

    B) Startup 커멘드 및 옵션들 작성하기

    ".Rprofile" 에 무언가를 작성할 것인데
    바로 이곳에 library(dplyr) 과 같은 것을 적고 저장시키면
    앞으로 R을 실행하자마자 library(dplyr) 명령어가 자동으로 실행된다.
    이러한 간단한 원리를 이용해 패키지 자동호출 외에도 options() set 을 미리 지정하는 것도 매우 좋은 Tip이 될 수 있겠다.

    나의 ".Rprofile" 을 소개하자면 이렇다.

    options(help_type = "html") ## 1
options(stringsAsFactors = FALSE) ## 2

pacman::p_load("useful.lovetoken", "knitr", "dplyr", "ggplot2") ## 3

opts_chunk$set(fig.path = "output/figure/", fig.align = "center", out.width = "80%", warning = F, message = F) ## 4

theme_set(theme_bw()) ## 5
    1. 함수 도움말을 볼 때 기본적으로 html 포맷의 형태에 문서가 열리는걸로 고정시켰다2
    2. read.csv() 와 같은 함수에서 stringsAsFactors 인자값이 getOption("stringsAsFactors") 에 의존되는데 이를 FALSE 로 고정시켰다
    3. 필자가 자주쓰는 패키지로 R을 실행할 때 무조건 호출될 패키지 4개를 pacman 을 통해 호출했다
    4. Rmarkdown 을 쓰는 경우 knitr 패키지를 통해 코드청크 글로벌 옵션들을 지정시켰다
    5. ggplot2 의 메인테마에 대한 설정이다

    여러분도 여러분 나름의 ".Rprofile" 를 작성해 보시길 🙃
    그리고 본 글은 Mac OS X 를 기준으로 작성한 경우이지만, 리눅스이어도 홈 디렉토리의 경로만 약간 다를 뿐 틸드문자를 이용한 방법은 똑같다고 보면 되겠다.
    윈도우의 경우는 R이 설치된 "C:\Program Files\R\R-***\etc\" 이곳에 ".Rprofile" 를 작성하면 되겠다.
